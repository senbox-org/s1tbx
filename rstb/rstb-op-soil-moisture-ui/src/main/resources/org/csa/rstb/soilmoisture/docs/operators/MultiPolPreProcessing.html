Multi-Pol Pre-Processing

Multi-Pol pre-processing takes a single C-band Quad Pol SLC product as input. 

The processing graph applies:

  • Radiometric calibration to Sigma0
  • Mean 3x3 speckle filtering
  • Terrain correction using the SRTM 3sec DEM and map projection to WGS84 Lat/Lng geographic coordinates

The multi-angle soil moisture retrieval requires that the terrain correction operator output the incidence angle from the ellipsoid as a band.

Pre-processing could also be applied from outside the Toolbox. In this case, the input product must adhere to the following input product conditions:

  1. The input product must be calibrated to Sigma0
  2. All required source bands, namely, sigma0HH, sigma0VV and Incidence Angle must be present in a single input source product.
  3. The sigma0HH band name should contain "Sigma0_HH" or "HH" with "Sigma0_HH" taking precedence meaning that if a band whose name contains "Sigma0_HH" is found, other bands whose name contains "HH" will be ignored.
  4. The sigma0VV band name should contain "Sigma0_VV" or "VV" with "Sigma0_VV" taking precedence.
  5. If no unit is specified for the sigma0 bands or the unit is not decibel (dB), the sigma0 values will be converted to dB.
  6. The incidence angle band could be local incidence angle (LIA) or incidence angles from the ellipsoid. Incidence angles should be in degrees.
  7. The incidence angle band name should contain "incidenceAngle" or "LIA" with "incidenceAngle" taking precedence.
